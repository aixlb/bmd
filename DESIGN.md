# bmd 架构设计

本文记录代码边界与必须长期保持的状态不变量。用户行为见 `MANUAL.md`，插件契约见 `PLUGINS.md`。

## 1. 分层

- `core/`：纯 CodeMirror 编辑内核。不得依赖 Vue、Pinia、Tauri 或应用 store。
- `src/components/`：界面与输入适配。组件调用 store action，不直接改写文件路径或跨模块状态。
- `src/stores/`：领域状态。`tabs` 管文档生命周期，`workspace` 管目录树，`files` 统一执行文件系统事务。
- `src/lib/ipc.ts`：Tauri 适配端口与浏览器/测试 mock。磁盘读写不得绕过此层。
- `src-tauri/`：文件系统、搜索、预览、AI、RAG 与导出实现。长时间文件扫描必须放入阻塞线程池。

## 2. 文档生命周期

每个标签 ID 在其生命周期内保持稳定。临时预览只复用一个标签槽位，异步加载使用递增请求号，只有最后一次用户意图可以提交。

可编辑标签维护 `revision` 与 `savedRevision`：

- 每次用户编辑递增 `revision`；`dirty = revision !== savedRevision`。
- 保存开始时捕获内容和修订号；写盘完成后只更新对应的 `savedRevision`，不得无条件清除 `dirty`。
- 同一标签的保存必须串行执行。窗口关闭和文件删除必须等待已经开始的保存结束。
- 磁盘写入采用唯一临时文件、`fsync` 和同目录替换；文本保存保持原始编码。

## 3. 文件系统事务

新建、重命名和删除统一经过 `useFiles`。目录操作按路径前缀影响所有内部标签：

- 重命名前保存受影响的脏文档；成功后同步映射标签路径并重算文件能力。
- 删除前等待在途保存；成功后统一关闭文件或目录下的标签。
- 组件不得直接修改 `tab.path`、`tab.title`、`tab.kind`。

文件能力规则位于 `shared/file-types.json`，前端与 Rust 共用。未知扩展会按内容探测文本，二进制内容必须拒绝进入文本编辑器。读取支持 UTF-8/BOM、UTF-16 与常见本地编码，写回保持原编码。

## 4. 会话与应用生命周期

会话只保存正式标签，不保存临时预览。启动恢复完成后立即写入一次；后续变化防抖写入；窗口销毁前必须显式 `flush()`。Tauri 事件监听、Vue watch、插件实例和计时器都必须在卸载时释放。

工作区切换引发的 AI 会话迁移由应用编排层负责，`workspace` 与 `ai` store 不得互相导入。store 不得依赖 Vue 单文件组件；菜单和对话框通过 `menuBus` 的纯类型端口注入。

## 5. 搜索

侧栏与 AI 搜索使用独立 scope 和单调请求号。新查询会取消同 scope 的旧查询；前端只接纳当前请求结果。Rust 递归扫描运行在阻塞线程池，并在遍历过程中检查取消状态。

## 6. 验证底线

每次涉及文档生命周期或文件操作的改动至少覆盖：

- 保存期间继续输入；连续保存；关闭时等待保存。
- 快速连续预览，只提交最后一次。
- 文件与目录重命名后继续编辑、保存和删除。
- 预览转正式后的会话快照与关闭前 flush。
- UTF-8、UTF-16、常见本地编码往返及二进制拒绝。

合入前运行 `npm test`、`npm run build`、`cargo test --manifest-path src-tauri/Cargo.toml` 与 `git diff --check`。
